Findizer

Findizer is a web application which allowing users to enjoy shopping services directly in their web browser. It can save your search, take some promotional code or discover the best product of the e-commerce.

Mission 1 : Launch the V3

Change The technical architecture and the graphics architecture. I had worked a lot on the technical part to integrate the MVC pattern on the website.

Technologies used :
  • HTML5/CSS3/JS
  • PHP/Mysql
Mission 2 : Web crawlers

To always have more products in the database, we need a continuous supply of products. And to accomplish that, we need platforms that pass us some formatted streams of hundreds different shops on the web. The stream is saved on the local webserver. The major problem was the size of stream. Because some of stream can make one giga-octet and contains more than 500 000 product, we need a strong, robust and timeless script to decode and save it in our database.

Technologies used :
  • PHP with gearman (to run mutliple instance of crawlers)
  • Mysql with temporary table and bulk insert/update to optimize the response time of database.
Mission 3 : Url matching (Lot of work on the algortihm part)

The "web trends" flow should allow the user to find the most popular products currently on the web. To do this, we therefore need to analyze the navigation of each plugins and try to match with our existing product base. We had approximately 100,000 active plugins that sent us, every day, more than a million URLs sailed on various e-commerce sites. We decided to use these URLs, compare with URLS products in our database and calculate a flow of "web trends".

Technologies used :
  • PHP/Mysql
Mission 4 : Internet Explorer plugin

Findizer already has a plugin for Firefox and Google Chrome. But in order to offer its services to a maximum of user, it is essential to offer a version for Internet Explorer that is still heavily used in the world of web browsers.

Technologies used :
  • C++
  • JS
Date : Septembre 2013 to February 2014
